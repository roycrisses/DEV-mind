## How to Submit a New Prompt

**Step 1 — Fork and clone the repo**
```bash
git clone https://github.com/YOUR_USERNAME/devmind.git
cd devmind
```

**Step 2 — Create your prompt file in the right category folder**
```bash
# Example: adding a new coding prompt
touch prompts/coding/my-new-prompt.md
```

**Step 3 — Write your prompt using the template below**

Every field is required. No exceptions.

**Step 4 — Test it yourself**

Before submitting, run your prompt in at least one AI (Claude, GPT-4o, Gemini, or Ollama) and confirm the output is actually useful. Paste a real example output.

**Step 5 — Open a Pull Request**

```bash
git checkout -b add-prompt/my-new-prompt
git add prompts/coding/my-new-prompt.md
git commit -m "feat(prompts): add [prompt name] to coding category"
git push origin add-prompt/my-new-prompt
```

Open a PR against `main`. Use the PR title format: `feat(prompts): add [prompt name] to [category]`

## The Prompt Template

Copy this exactly and fill in every section:

```markdown
---
title: [Prompt Name]
category: [coding | devops | docs | testing | git]
version: 1.0.0
works_with: [claude, openai, ollama, gemini]
---

## Purpose
One sentence describing what this prompt does.

## When to Use
- Bullet 1: exact situation
- Bullet 2: exact situation
- Bullet 3: exact situation

## The Prompt

\```
[Your prompt text here. Use {{VARIABLE_NAME}} for all dynamic parts.
Be specific. Be complete. Write it like you're briefing a contractor.]
\```

## Example Input

| Variable | Value |
|---|---|
| `{{VARIABLE_NAME}}` | A realistic example value |

## Example Output

\```
[Paste a real AI-generated response you got from this prompt.
Do NOT make one up.]
\```

## Tips
- Tip 1: something specific about getting better results
- Tip 2: a model-specific tweak
- Tip 3: a common mistake to avoid
```

## Quality Standards

### ✅ What Gets Merged

- Prompt is genuinely useful to working developers
- All `{{VARIABLES}}` are clearly named and documented
- Example output is real (actually generated by an AI, not invented)
- The prompt works across at least 2 providers
- Frontmatter is fully filled in
- File is named with kebab-case (`my-prompt-name.md`)
- Adds unique value — not a duplicate of an existing prompt

### ❌ What Gets Rejected

- Generic prompts that any developer would already think of
- Missing or fake example outputs
- Prompts with no `{{VARIABLES}}` (they're not reusable)
- Files with broken markdown or missing frontmatter
- Prompts that only work with one specific provider
- Vague purpose statements ("helps with coding")

## Suggesting a New Category

If you think there's a whole category missing (e.g., `security/`, `architecture/`, `database/`), open a GitHub Issue with:

1. **Category name** and the folder path (e.g., `prompts/security/`)
2. **At least 3 concrete prompt ideas** that would live in it
3. **Why it doesn't fit** in any existing category

We'll discuss in the issue before you create any files.

## Code of Conduct Summary

DevMind follows the [Contributor Covenant](https://www.contributor-covenant.org/version/2/1/code_of_conduct/).

- Be respectful and constructive in all interactions
- No harassment, discrimination, or personal attacks
- Assume good intent; ask questions before assuming malice
- Violations can be reported to the maintainers via GitHub Issues (mark as `[CoC]` in the title)

Repeated violations result in a permanent ban from the repo.
